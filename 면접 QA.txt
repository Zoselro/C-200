dbus란?
자세히는 잘 모르지만, 제가 배웠던 바로는
dbus란 DataBus로 주로 통신 IPC를 위해 메시지 버스 시스템 이며,
모듈끼리 복잡한 프로그래밍 필요없이 간단하게 통신하여 값을 주고 받고 할 수 있음.
특정 모듈로부터 값을 sync(동기식) 방식으로 받아오면서 그 값을 다른 모듈에서
함수의 파라미터로 지정하여 유연하게 그 값으로 처리 할 수 있다.

상세 업무 능력(업무 성과 및 태도 포함)
개발 기간이 8개월이라 어렵고 복잡한 이슈는 받진 않았다. -> 쉬운 이슈인줄 알고 보니까 어려운 이슈였다. -> 팀원들과 분석을 하고, 주말까지 출근하면서 협업하여 이 이슈를 해결하였다.

저는 해당 회사에서 8개월 동안 네비게이션 시스템 개발에 참여하며 다양한 기술적 도전을 경험했습니다. 
이 기간 동안 복잡한 이슈는 많지 않았지만, 기억에 남는 사례로는 특정 지역에서 네비게이션 지도에 위치 
정보가 튀는 현상에 대한 수정 요청이 있었습니다.
이 문제를 해결하기 위해 먼저 코드를 면밀히 분석하고 모의 테스트를 진행했습니다. 
초기 테스트에서는 특별한 문제가 발견되지 않았으나, 고객으로부터 지속적으로 위치 튐 현상이 발생한다는 피드백이 들어왔습니다.
 이를 해결하기 위해 팀원들과 며칠 동안 주행 테스트를 진행하며 데이터를 수집하고, 로그를 분석했습니다.

그 결과, 특정 터널이나 높은 고가도로를 지나갈 때 위치 정보가 튀는 현상이 발생한다는 사실을 확인했습니다. 
추가적으로 Dubs 통신을 통해 Gyro 센서의 상태를 점검한 결과, 교정(calibration) 측정값이 100%일 때도 Auto 모드가 
아닌 Manual 모드로 설정되는 문제가 있음을 발견했습니다. 이로 인해 위치 정보의 정확성이 저하되는 것을 확인한 후, 코드를
 수정하여 문제를 해결했습니다.
이 경험을 통해 문제 분석과 팀워크의 중요성을 다시 한번 깨닫게 되었고, 실제 차량 테스트를 통해 데이터의 신뢰성을 높이는 과정에서
 큰 보람을 느꼈습니다. 앞으로도 이러한 문제 해결 능력을 바탕으로 지속적인 성장을 이루고, 회사에 기여할 수 있는 개발자가 되고자 합니다.

Q : 자동차의 HU는 컴퓨터로 따지면 뭘까?
A : 자동차의 HU(Head Unit)는 컴퓨터에서의 중앙처리장치(CPU)와 유사한 역할을 합니다.
 HU는 차량의 인포테인먼트 시스템과 관련된 여러 기능을 관리하고 제어하는 장치로, 
 오디오, 내비게이션, 통신, 차량 상태 모니터링 등의 기능을 통합합니다. 
 즉, HU는 차량의 다양한 전자 시스템을 통합하고 사용자 인터페이스를 제공하는 핵심적인
  컴퓨터 시스템으로 볼 수 있습니다.

초기에는 C++ 기본 문법과 프로젝트 진행 방식을 익히기 위해 3개월간의 세미나를 통해 기술적 이해를 쌓았고, 이후 Git을 통한 형상 관리 방식을 익히면서 프로젝트에 기여할 수 있었습니다.

다만, 네이티브 애플리케이션 개발에 집중되다 보니 코드 변경 사항을 빌드하고 ipk 설치 후 검증해야 문제를 확인할 수 있었습니다. 이러한 과정에서는 변경 사항을 즉시 반영하고 테스트하기 어려워 아쉬움을 느끼기도 했습니다.
그렇기에 코드를 살피면서 이슈가 재발생 되지 않게끔 정밀하게 검토를 진행을 하면서 개발 실력을 키워 나갔습니다.

또한, Git을 활용한 실무 경험과 Linux 환경에서 작업을 하다보니, Linux Command 학습을 통해 협업 환경에서 업무 처리 능력을 키웠습니다. 이러한 경험을 바탕으로, C와 같은 절차지향 언어 뿐만이 아니라 C++ , Java 처럼 객체지향 언어에서의 개발 역량을 더욱 성장시키고, 시스템을 구축할 수 있는 개발자로서 새로운 도전에 나서고자 지원하게 되었습니다.
입사 후 포부
≪≪ 비전과 기여 계획 ≫≫

현재 게임 관련 API 데이터를 python과 연동을 한 후 그 데이터를 javascript인 Ajax를 사용하여 데이터를 요청한 후 그 데이터를 DB로 데이터를 가공하고, spring을 통해 데이터를 응답시키는 프로젝트를 하고 있는 중 입니다. 언어를 두 가지 사용하는 이유는 다양한 언어를 다루는 경험을 통해 기술 스택을 넓히고, 각 언어와 기술의 강점을 활용하여 더욱 효율적인 시스템을 구축하기 위함 입니다. 이를 통해 프로젝트의 요구사항에 맞춘 최적의 솔루션을 제공하고, 기술 간의 융합을 통해 시너지를 창출하는 개발자가 되고자 합니다. 입사 후, 안정적이고 확장 가능한 시스템을 구축하는 데 기여하고 싶습니다.

또한, GPS 관련 시스템 개발 경험을 바탕으로 복잡한 이슈를 분석하고 해결하는 데 있어 적극적인 역할을 수행하겠습니다. 더불어,
Git을 통한 협업 환경에서의 효율적인 코드 관리와 Linux 명령어를 활용한 서버 운영 경험을 바탕으로 팀의 성과를 극대화하겠습니다.
지속적인 학습과 자기 개발을 통해 회사의 성장과 함께 발전하는 개발자로 자리매김하겠습니다.
상세 업무 능력
저는 해당 회사에서 8개월 동안 네비게이션 GPS 관련 모듈 개발을 담당하며 다양한 기술적 도전을 경험했습니다. 이 기간 동안 복잡한 이슈는 많지 않았지만, Linux 와 git을 기본으로 쓰기에 grep, find와 같이 기본적인 command 와 git에서 add commit push 같은 기본적인 command 부터 시작해서 심화적인 command 까지의 명령어를 개인적으로 익히면서 공부를 하며 jira를 통해 다른분들의 issue를 Tracking 하면서 어떤 이슈가 발생이 되는지에 알아봤습니다. 제 이슈중에서 기억에 남는 사례로는 특정 지역에서 네비게이션 지도에 위치 정보가 튀는 현상에 대한 수정 요청이 있었습니다. 이 문제를 해결하기 위해 먼저 코드를 면밀히 분석하고 모의 테스트를 진행했습니다.

초기 테스트에서는 특별한 문제가 발견되지 않았으나, JIRA를 통해 협업사로부터 지속적으로 위치 튐 현상이 발생한다는 피드백이 들어왔습니다.
이를 해결하기 위해 grep 명령어를 통해 어떤 함수가 문제가 되는지에 찾아보고, 이후 팀원들과 며칠 동안 주행 테스트를 진행하며 데이터를 수집하고, 로그를 분석했습니다.
그 결과, 특정 위치에서 이슈가 발생한다는 사실을 확인했습니다. 추가적으로 Dubs 통신을 통해 센서의 상태를 점검을 통해 더욱더 명확한 이슈 원인을 확인을 하였고, 이로 인해 위치 정보의 정확성이 저하되는 것을 확인한 후, 코드를 수정하여 문제를 해결했습니다.

이 경험을 통해 문제 분석과 팀워크의 중요성을 다시 한번 깨닫게 되었고, 실제 차량 테스트를 통해 데이터의 신뢰성을 높이는 과정에서 큰 보람을 느꼈습니다. 앞으로도 이러한 문제 해결 능력을 바탕으로 지속적인 성장을 이루고, 회사에 기여할 수 있는 개발자가 되고자 합니다.
